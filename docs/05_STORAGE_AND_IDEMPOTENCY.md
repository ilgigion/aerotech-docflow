# 06. Архив, атомарность и идемпотентность

## Финальное имя

```text
ТИП_ГГММДД_ЧЧММСС_НОМЕР.pdf
```

Пример:

```text
УПД_260710_101025_2455B.pdf
```

## Архив

```text
ARCHIVE_ROOT\ГОД\ТИП\filename.pdf
```

## `.tmp`

Файл сначала копируется во временный `.tmp` рядом с финальным местом и проверяется.
Финальное имя публикуется атомарным hard link без overwrite, после чего временное
имя удаляется. Если финальный путь уже существует, существующий PDF не изменяется,
а storage возвращает контролируемую ошибку.

## `.reserve`

Перед переносом резервируется финальное имя через `.reserve`. Это защищает от гонки, когда два процесса пытаются выбрать одно и то же имя.

## Идемпотентность

Если передан `idempotency_key`, создаётся JSON-маркер в:

```text
D:\incoming\_idempotency
```

Поведение:

```text
succeeded → вернуть существующий результат без нового скана
scanned   → повторить только storage
processing/storing → операция уже выполняется
failed/interrupted/timeout → разрешить новую попытку
```

После резервирования имени статус `storing` содержит также точные
`final_file_name` и `final_file_path`. Они записываются до копирования и
атомарной публикации. Если процесс завершился после публикации PDF, но до
удаления файла из incoming или записи `succeeded`, повторная попытка:

1. проверяет, что сохранённый final path находится внутри `ARCHIVE_ROOT`;
2. строго проверяет структуру PDF;
3. при наличии исходника сравнивает размер и SHA-256 двух файлов;
4. принимает уже опубликованный файл и не создаёт дубликат `_01`.

Если опубликованный и временный PDF различаются, автоматическое восстановление
останавливается с `idempotency_published_file_mismatch`; оба файла сохраняются
для ручной проверки.

Если `idempotency_key` не передан, каждый запуск считается новой операцией.

Один `idempotency_key` связан с fingerprint параметров задачи и документа. Повтор
того же ключа с другими параметрами отклоняется. Пути из JSON принимаются только
внутри настроенных incoming/archive каталогов; PDF повторно проходит строгую
проверку перед replay или retry storage.
