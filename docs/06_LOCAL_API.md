# Локальный API

Минимальный FastAPI-сервер предоставляет два endpoint:

- `GET /health` — проверка процесса без обращения к сканеру;
- `POST /scan` — синхронный запуск существующего `document_flow`.

Сервер привязан только к `127.0.0.1:8000`. В нём нет очереди, worker, polling,
Planfix-интеграции, API key/HMAC и внешнего туннеля. Не публикуйте этот процесс
на внешнем интерфейсе без отдельного защищённого шлюза.

## Установка и запуск

```powershell
pip install -r requirements.txt
python -m app.run_local_api
```

Проверка состояния:

```powershell
curl http://127.0.0.1:8000/health
```

Запрос на сканирование:

```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:8000/scan" `
  -ContentType "application/json" `
  -Body '{
    "task_id": "53243",
    "doc_type": "НКЛ",
    "document_datetime": "2026-06-24T13:50:00",
    "document_number": "001",
    "idempotency_key": "planfix_53243_НКЛ_20260624T135000_001"
  }'
```

`idempotency_key` можно не передавать. Тогда сервер стабильно формирует его из
`task_id`, нормализованного типа документа, даты/времени и нормализованного номера.
Повторный успешный запрос с тем же ключом возвращает сохранённый результат с
`scan_executed: false`.

Успешный ответ содержит `file_name`, но намеренно не раскрывает локальный
`file_path`. Занятый сканер и выполняющаяся идемпотентная операция дают HTTP 409,
ошибка входных данных — 422; timeout, недоступный сетевой сканер или
необходимость ручного восстановления — 503; остальные ошибки
сканирования/хранилища — 500.

`operation_id` в HTTP-ответе всегда создаётся до входа в document flow и
передаётся во все стадии. Поэтому при контролируемой ошибке идентификатор в
ответе совпадает с идентификатором в scanner/storage/idempotency-логах.

Настройки NAPS2, входной директории и архива загружаются самим `document_flow` из
существующих переменных окружения и значений по умолчанию проекта.

В будущем локальный endpoint может вызываться ПланФиксом через отдельный внешний
HTTPS-туннель. Такой туннель и его аутентификация не входят в текущую реализацию.

## Ручные команды

Эти команды не запускаются unit-тестами автоматически:

```powershell
python -m app.run_local_api
python -m tests.manual.run_scan_epson_profile
python -m tests.manual.run_scanner_recovery_diagnostics
```
