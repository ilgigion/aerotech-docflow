# Release candidate: исправления по приёмочным испытаниям 2026-07-16

## Исправленные дефекты

### Дубликат после аварии между публикацией PDF и фиксацией успеха

До исправления процесс мог атомарно опубликовать финальный PDF, завершиться до
удаления incoming-файла и оставить idempotency в `storing`. Повтор storage
создавал идентичный файл с суффиксом `_01`.

Теперь точный final path записывается в idempotency сразу после резервирования,
до копирования. При восстановлении уже опубликованный PDF строго проверяется и,
если исходник сохранился, сравнивается с ним по размеру и SHA-256. Совпавший
файл принимается без повторной публикации; несовпадение останавливает автоматику.

### Несовпадающий `operation_id` в ответе и логах

Безопасная обёртка раньше создавала fallback-ID отдельно от document flow.
Теперь один ID создаётся до запуска flow и передаётся во все стадии, поэтому
ошибка, HTTP-ответ и логи имеют единый идентификатор.

### Сетевые ошибки NAPS2 определялись как пустой податчик

Классификация сетевых маркеров выполняется до общей строки
`No scanned pages to export`. Подтверждённые сообщения SSL, некорректного
сетевого ответа и отключённого сканера возвращают `scanner_connection_error`;
реальный пустой податчик остаётся `no_scanned_pages`.

### Localhost-запросы приёмки зависели от VPN/proxy

Acceptance runner использует `httpx.Client(trust_env=False)`. Переменные
`HTTP_PROXY` и `HTTPS_PROXY` больше не могут направить запрос к
`127.0.0.1:8000` через внешний прокси.

### Невоспроизводимые вспомогательные сценарии

Каждый новый acceptance run теперь содержит ASCII-совместимый с Windows
PowerShell 5.1 сценарий серии из 20 документов. Пустой Enter не запускает
сканирование: требуется явное слово `SCAN`. Проверка серии автоматически ищет
baseline текущего run и не содержит путей от предыдущего испытания.

## Автоматическая проверка

Выполнены без NAPS2 и физического сканера:

```powershell
python -m compileall -q app tests
python -m tests.unit.run_all_unit_tests
python -m tests.acceptance.run_acceptance_tests start --runs-root <TEMP_DIR>
```

Результат:

- compileall — пройден;
- полный unit suite — пройден;
- storage-пробы сценариев 7, 8 и 9 — пройдены;
- сгенерированный PowerShell-файл — ASCII и успешно разобран Windows
  PowerShell 5.1.
