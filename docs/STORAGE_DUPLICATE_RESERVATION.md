# 3.4. Защита от дублей при параллельной работе

## Проблема

Даже если есть логика:

```text
если файл существует → добавить _01, _02, ...
```

остаётся гонка:

```text
процесс A проверил: файла нет
процесс B проверил: файла нет
оба выбрали одно имя
оба начали перенос
```

Сейчас общий `scanner.lock` уже сильно снижает риск, потому что полный процесс сканирования выполняется по одному. Но для будущей очереди + worker лучше дополнительно защитить сам выбор финального имени.

---

## Решение

Добавлен механизм резервирования финального имени через `.reserve`-файл.

Пример финального файла:

```text
УПД_260710_101025_2455B.pdf
```

Технический файл резервирования:

```text
.УПД_260710_101025_2455B.pdf.reserve
```

Он создаётся атомарно через:

```python
os.O_CREAT | os.O_EXCL
```

Это означает: создать файл только если его ещё нет.

---

## Схема работы

```text
1. storage.py формирует базовое имя файла.
2. Проверяет, существует ли final.pdf.
3. Проверяет, существует ли .final.pdf.reserve.
4. Если оба свободны — атомарно создаёт .reserve.
5. Копирует PDF во временный .tmp.
6. Проверяет .tmp.
7. Переименовывает .tmp в final.pdf.
8. Удаляет .reserve.
9. Удаляет source из incoming.
```

Если имя занято, выбирается следующий вариант:

```text
УПД_260710_101025_2455B.pdf
УПД_260710_101025_2455B_01.pdf
УПД_260710_101025_2455B_02.pdf
```

---

## Что происходит при сбое

Если ошибка произошла до финального переименования:

```text
source остаётся в incoming
.tmp удаляется
.reserve удаляется
final.pdf не создаётся
```

Если другой процесс уже зарезервировал имя:

```text
текущий процесс выбирает _01, _02, ...
```

Если `.reserve` старый, его можно удалить автоматически.

Настройка:

```python
StorageSettings(
    reservation_stale_after_seconds=30 * 60,
)
```

## Как тестировать

Обычный тест дублей:

```powershell
python -m tests.run_storage_reservation_test
```

Тест искусственной коллизии `.reserve`:

```powershell
python -m tests.run_storage_reservation_collision_test
```

Ожидаемо:

```text
первый файл получает базовое имя
второй файл получает _01
.reserve и .tmp после успешной операции не остаются
```
