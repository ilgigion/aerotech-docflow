# 1. Назначение и архитектура

## Что делает программа

Aerotech Docflow принимает метаданные документа по локальному HTTP API,
запускает NAPS2, получает PDF, строго проверяет его и безопасно публикует в
структуру архива:

```text
D:\REPLACE_WITH_ARCHIVE_ROOT\ГОД\ТИП\ТИП_ГГММДД_ЧЧММСС_НОМЕР.pdf
```

Программа решает четыре основные задачи:

1. не запускать один физический сканер параллельно;
2. не принять пустой или повреждённый PDF как успешный;
3. не перезаписать существующий файл архива;
4. позволить безопасно повторить запрос после сетевого или программного сбоя.

## Из чего состоит система

```text
клиент / Postman / локальный интегратор
                  │
                  │ POST /scan
                  ▼
            FastAPI: app.api
                  │
                  ▼
      валидация запроса и business rules
                  │
                  ▼
       файловая идемпотентность JSON
                  │
                  ▼
        глобальный lock физического сканера
                  │
                  ▼
      NAPS2.Console.exe + профиль сканера
                  │
                  ▼
             C:\ProgramData\Aerotech Docflow\incoming\PF_*.pdf
                  │
                  ▼
       строгая проверка PDF через pypdf
                  │
                  ▼
       резервирование имени через .reserve
                  │
                  ▼
       копирование в соседний скрытый .tmp
                  │
                  ▼
     размер + SHA-256 + повторная PDF-проверка
                  │
                  ▼
      атомарная публикация без overwrite
                  │
                  ▼
          D:\REPLACE_WITH_ARCHIVE_ROOT\ГОД\ТИП\*.pdf
```

## Полный цикл одного запроса

1. FastAPI разбирает JSON.
2. Pydantic проверяет обязательные поля, длины и отсутствие лишних полей.
3. Приложение проверяет канонические `task_id`, `doc_type`,
   `document_number` и безопасное точное имя `scanner_profile`.
4. Production-правила проверяют разрешённый тип документа.
5. Создаётся `operation_id`, например `SCAN_20260721_120000_a1b2c3`.
6. Определяется `idempotency_key`: переданный клиентом либо вычисленный из
   реквизитов.
7. Читается или атомарно создаётся JSON идемпотентности.
8. Если документ уже успешно сохранён, возвращается существующий результат без
   сканирования.
9. Для новой операции создаётся `C:\ProgramData\Aerotech Docflow\incoming\.scanner.lock`.
10. Непосредственно перед запуском NAPS2 сервер фиксирует время `Europe/Moscow`,
    проверяет год и формирует имя `ТИП_ГГММДД_ЧЧММСС_НОМЕР.pdf`.
11. NAPS2 создаёт уникальный `PF_*.pdf` во временной папке.
12. PDF проверяется по размеру, заголовку, EOF, структуре и числу страниц.
13. Резервируется свободный путь в архиве.
14. PDF копируется, сверяется по SHA-256 и публикуется атомарным hard link.
15. JSON идемпотентности получает статус `succeeded`, исходный временный PDF
    удаляется, lock освобождается, API возвращает ответ.

## Почему API синхронный

`POST /scan` держит HTTP-соединение открытым до завершения сканирования и
архивирования. Отдельной очереди и фонового worker нет. Поэтому:

- клиент сразу получает окончательный результат;
- второй физический скан не запускается параллельно;
- долгий скан означает долгий HTTP-запрос;
- клиентский timeout должен быть больше `scanner.timeout_seconds` с запасом.

## Границы ответственности

Приложение гарантирует безопасность операций, которые проходят через его код.
Оно не может предотвратить:

- ручное удаление PDF администратором;
- повреждение диска или файловой системы;
- выключение питания в произвольный момент;
- изменение архива сторонней программой;
- сетевой разрыв или отказ прошивки сканера;
- запуск NAPS2 другим пользователем одновременно вне приложения.

Эти случаи выявляются через логи, контрольные суммы, lock/reservation и
приёмочные испытания.

## Что не входит в текущую версию

- публичный API;
- API key, HMAC, OAuth или пользовательская авторизация;
- автоматический SSH-туннель;
- очередь нескольких сканов;
- OCR;
- Planfix-логика;
- автоматическое обновление приложения;
- автоматическое удаление неизвестных аварийных файлов.
