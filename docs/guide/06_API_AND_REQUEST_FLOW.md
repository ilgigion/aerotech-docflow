# 6. HTTP API и полный цикл запроса

## Адрес и границы доступа

```text
http://127.0.0.1:8000
```

Приложение разрешает bind только на `127.0.0.1` или `localhost`. В текущей
версии API не имеет собственного логина, токена или HMAC. Это допустимо только
потому, что порт локальный. Не меняйте host на `0.0.0.0` и не публикуйте порт
напрямую в интернет.

Если в будущем будет использоваться SSH-туннель, авторизация, TLS и ограничение
источника должны обеспечиваться внешним защищённым слоем. Это не часть текущего
приложения.

## `GET /health`

Назначение: проверить, что процесс FastAPI запущен.

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health
```

Ответ:

```json
{
  "status": "ok",
  "service": "aerotech-docflow",
  "scanner_api": "local",
  "version": "1.0.0-rc1+b45d4c5f"
}
```

Endpoint не обращается к NAPS2, не захватывает lock и не пишет PDF.

## `POST /scan`

### Схема входа

| Поле | Тип | Обязательное | Ограничения |
|---|---|---|---|
| `task_id` | string | да | 1–120 символов, каноническое file-safe значение |
| `doc_type` | string | да | 1–30, uppercase, разрешён production allowlist |
| `document_number` | string | да | 1–80, без неоднозначных символов имени файла |
| `scanner_profile` | string | да | 1–200, точное имя профиля NAPS2, без управляющих символов |
| `idempotency_key` | string/null | нет | 1–500, стабильный ключ одной бизнес-операции |

Лишние поля запрещены. Пробелы по краям строк удаляются. `/`, `\\`, опасные
или lossy-нормализуемые значения не исправляются молча, а отклоняются с 422.

### Полный запрос

```json
{
  "task_id": "53243",
  "doc_type": "НКЛ",
  "document_number": "001",
  "scanner_profile": "MY_NAPS2_PROFILE",
  "idempotency_key": "planfix_53243_НКЛ_001"
}
```

Поле `document_datetime` передавать нельзя: модель запрещает лишние поля и
вернёт 422. Дата и время имени фиксируются самим сервером в `Europe/Moscow`
непосредственно перед запуском NAPS2, после захвата scanner lock. Поэтому время
ожидания занятого сканера не попадает в имя.

Если `idempotency_key` отсутствует, API формирует его как:

```text
planfix_TASK_ТИП_НОМЕР
```

Для интеграции лучше передавать ключ явно и хранить его вместе с задачей.

`scanner_profile` выбирает профиль для этого конкретного физического запуска.
Имя передаётся NAPS2 как отдельный аргумент `-p`, поэтому кавычки shell не
интерпретируются. Путь к NAPS2, incoming, timeout и проверки PDF по-прежнему
берутся из `config.toml`. Имя профиля участвует в fingerprint: менять его при
том же `idempotency_key` нельзя — сервер вернёт 409. При идемпотентном replay
NAPS2 вообще не запускается, что видно по `scan_executed: false`.

### Успешный ответ

```json
{
  "status": "succeeded",
  "file_name": "НКЛ_260721_135102_001.pdf",
  "operation_id": "SCAN_20260721_135102_a1b2c3",
  "task_id": "53243",
  "idempotency_key": "planfix_53243_НКЛ_001",
  "scan_executed": true
}
```

Локальный `file_path` намеренно не возвращается, чтобы не раскрывать структуру
дисков внешнему клиенту. Путь однозначно определяется по конфигу, году, типу и
`file_name`.

### Ошибочный ответ

```json
{
  "status": "failed",
  "error_code": "no_scanned_pages",
  "message": "Сканирование завершилось, но страницы не были получены...",
  "operation_id": "SCAN_20260721_135102_a1b2c3",
  "task_id": "53243",
  "idempotency_key": "planfix_53243_НКЛ_001"
}
```

`operation_id` создаётся до document flow. Он должен совпадать в HTTP-ответе,
scanner/storage/idempotency и application logs.

## HTTP-коды

| Код | Значение | Типичные причины |
|---:|---|---|
| 200 | Документ сохранён либо возвращён идемпотентный результат | `succeeded` |
| 409 | Контролируемый конфликт | сканер/lock занят, операция с ключом уже идёт, ключ использован для другого документа |
| 422 | Неверный запрос | тип поля, лишнее поле, неканоническое имя, запрещённый тип |
| 503 | Временная недоступность или ручное восстановление | сеть сканера, timeout, NAPS2 не остановился, poisoned-state |
| 500 | Контролируемая ошибка scanner/storage либо внутренняя ошибка | битый PDF, недоступный архив, ошибка копирования |

HTTP 500 не означает, что архив обязательно повреждён. Система возвращает 500 и
останавливается именно для того, чтобы не заявить ложный успех.

## Как связать запрос с файлом

Сохраняйте вместе:

```text
task_id
idempotency_key
operation_id
file_name
HTTP status
timestamp
```

По `operation_id` ищется техническая история. По `idempotency_key` — состояние
бизнес-операции и повтор. По `file_name` — конечный PDF.

## Postman

В Postman:

1. Method: `POST`;
2. URL: `http://127.0.0.1:8000/scan`;
3. Body → raw → JSON;
4. Header `Content-Type: application/json`;
5. timeout больше scanner timeout с запасом.

При timeout Postman сначала проверьте логи и повторите тот же ключ. Не создавайте
новый запрос с новым ключом, пока неизвестен результат первого.
