# 14. Термины

## API

Программный HTTP-интерфейс. В проекте это локальные `/health` и `/scan`.

## Archive root

Корень выбранного архива, например `D:\REPLACE_WITH_ARCHIVE_ROOT`. Программа создаёт под ним
папки года и типа, но не должна создавать root без административного решения.

## Atomic / атомарный

Операция наблюдается как целиком выполненная либо не выполненная. Для final PDF
используется hard link без overwrite.

## ADF / автоподатчик

Automatic Document Feeder — устройство подачи нескольких листов сканера.

## CLI

Командный интерфейс `aerotech-docflow.exe`: `show-config`, `preflight`,
`diagnose`, `run`.

## Direct mode

Запуск NAPS2 без GUI-профиля через явные driver/device/source параметры.

## Fail-closed

При сомнении система отказывает и сохраняет данные для проверки, вместо
попытки угадать и сообщить ложный успех.

## Fingerprint запроса

Стабильное представление реквизитов документа, привязанное к idempotency key.
Защищает один ключ от использования для другого документа.

## Hard link

Второе имя тех же данных внутри файловой системы. Создание final hard link
атомарно и не перезаписывает существующий путь.

## Idempotency / идемпотентность

Свойство повторного запроса возвращать тот же результат без повторного
физического действия. Реализовано JSON-файлами состояния.

## Incoming

Локальная временная папка, куда NAPS2 пишет `PF_*.pdf`. Это не архив.

## Lock

Файл владения физическим сканером. Пока lock активен, второй скан запрещён.

## Marker

`.aerotech-docflow-archive.json` с identity архива. Защищает от случайного
переключения на похожий путь.

## NAPS2

Внешняя программа, которая работает с драйвером сканера и создаёт PDF.
Docflow запускает `NAPS2.Console.exe`.

## Operation ID

Уникальный технический идентификатор одной попытки. Связывает HTTP, scanner,
storage, idempotency и logs.

## PDF validation

Проверка размера, заголовка, EOF, структуры через `pypdf` и числа страниц.

## Poisoned-state

Состояние, в котором автоматическое продолжение может причинить вред: например,
NAPS2 не остановился или temp/final различаются. Требуется ручная проверка.

## Preflight

Проверка production-конфигурации и путей без запуска сканера и записи PDF.

## Profile NAPS2

Пользовательская конфигурация устройства, источника, duplex, качества и других
параметров. Принадлежит конкретному Windows-пользователю.

## Reservation / `.reserve`

Атомарная бронь конкретного final name. Содержит operation, PID, hostname,
destination и время.

## Replay

Возврат уже подтверждённого `succeeded` без нового скана (`scan_executed=false`).

## Retry storage

Повтор архивирования уже существующего валидного `PF_*.pdf` без физического
сканирования.

## SHA-256

Криптографическая контрольная сумма. Используется для доказательства
неизменности и совпадения source/temp/final.

## Stale

Артефакт достаточно старый и его процесс-владелец не жив. Только сочетание
времени и проверки PID позволяет автоматический takeover.

## TOML

Формат рабочего `config.toml`. Секции записываются как `[scanner]`, значения —
как `key = value`.

## WinSW

Windows Service Wrapper. Регистрирует EXE как службу, управляет стартом,
учётной записью, рестартами и wrapper logs.
