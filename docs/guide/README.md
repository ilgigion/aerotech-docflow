# Полное руководство Aerotech Docflow

Это основная документация по установке, эксплуатации и сопровождению Aerotech
Docflow. Она рассчитана на человека, который раньше не работал с проектом и не
должен обращаться к разработчику для выполнения штатных операций.

Исходная версия находится в `docs\guide`. При сборке она копируется в
`dist\AerotechDocflow\docs\guide`, а после новой установки — в
`C:\Program Files\Aerotech Docflow\docs\guide`.

## Обозначения безопасности

В тексте используются три типа действий:

- **только чтение** — не запускает сканер и не изменяет архив;
- **изменение конфигурации** — меняет настройки, но не создаёт PDF;
- **физический скан** — запускает NAPS2 и может создать документ в архиве.

Команды `show-config`, `preflight`, `diagnose` и `GET /health` не запускают
сканер. Команда `POST /scan` запускает физическое сканирование, если запрос не
является успешным идемпотентным повтором.

## Текущая поддерживаемая схема

Руководство описывает установленную сборку `onedir`:

```text
C:\Program Files\Aerotech Docflow\
C:\ProgramData\Aerotech Docflow\
C:\ProgramData\Aerotech Docflow\incoming\
D:\REPLACE_WITH_ARCHIVE_ROOT\
```

API слушает только `127.0.0.1`. В текущей версии нет публичного HTTPS,
авторизации API, очереди заданий, Planfix-интеграции и автоматической установки
драйвера сканера.

## Главное правило

Никогда не удаляйте вручную PDF, `.scanner.lock`, `.reserve`, `.tmp` или JSON
идемпотентности только потому, что они выглядят старыми. Сначала выполните
диагностику и установите владельца артефакта. Система специально предпочитает
остановиться, а не рисковать существующим архивом.
